Detailed Description

The flexible_type_record represents a collection of (field-id, flexible_type) pairs.

The internal represention of the flexible type record varies depending on the number of fields it contains. When it contains just a single field, it is a single 16-byte entry which represents a flexible_type with the field information held in the tag in the flexible_type. When the number of fields is larger than one, it maintained a pair of (size_t, flexible_type*) which is also 16 bytes, and represents a length, and a pointer to an array of flexible_type objects. Once again the field id is maintained in the tag in the flexible_type.
